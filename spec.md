# spec.md — 経費管理アプリ 仕様書

## 1. 目的・利用シーン

### 目的

ひとり用の経費管理アプリ。紙のレシートや表計算ソフトで行っていた経費記録を、ブラウザ1枚で完結させる。業務用SaaSが持つCRUD機能（登録・閲覧・検索・編集・削除）の核心部分を、ローカル動作・ゼロインストールで再現する。

### 利用シーン

- 出張・外出後にレシートを見ながら経費を登録する
- 月末に申請済み経費をまとめて「精算済」に切り替える
- 過去の交通費や接待費をキーワードで検索して確認する
- 経費の合計金額・件数をサマリで素早く把握する

---

## 2. データ項目

| 項目名 | キー | 必須 | 型 | 備考 |
|---|---|---|---|---|
| ID | `id` | ○ | string | `"exp_" + Date.now()` で自動生成 |
| 日付 | `date` | ○ | string | 経費発生日。`YYYY-MM-DD` 形式 |
| カテゴリ | `category` | ○ | string | 交通費／会議費／接待費／消耗品／通信費／その他 |
| 金額 | `amount` | ○ | number | 円単位の正の整数。0円不可 |
| メモ | `memo` | - | string | 用途・訪問先など自由記述。空文字列可 |
| ステータス | `status` | ○ | string | `"pending"`（申請中）or `"settled"`（精算済）。登録時は `"pending"` 固定 |
| 作成日時 | `createdAt` | ○ | string | `new Date().toISOString()` で自動付与。ユーザー編集不可 |

## 3. 画面構成

### 全体レイアウト

PC表示専用のツーカラム構成。左ペインが固定幅、右ペインが残余幅を占有する。

```
┌─────────────────────┬────────────────────────────────┐
│   左ペイン（380px）  │     右ペイン（flex-1）          │
│                     │                                │
│  [サマリカード]      │  [空状態 / 詳細 / フォーム]    │
│  [検索 | +新規]      │                                │
│  ─────────────────  │                                │
│  [経費カード]        │                                │
│  [経費カード]        │                                │
│  [経費カード]        │                                │
│  ...                │                                │
└─────────────────────┴────────────────────────────────┘
```

### 左ペイン

| 要素 | 内容 |
|---|---|
| サマリカード | 合計金額・申請中件数・精算済件数を常時表示。経費の追加・削除・ステータス変更に連動してリアルタイム更新 |
| 検索ボックス | メモ・カテゴリを対象にリアルタイムフィルタリング |
| 新規登録ボタン | クリックで右ペインをフォームモードに切替 |
| 経費カードリスト | 作成日時の新しい順に並ぶ。検索中はヒット件数のみ表示 |

#### 経費カードに表示する情報

- 日付（`YYYY/MM/DD` 形式）
- 金額（`¥XX,XXX` 形式）
- カテゴリ
- メモの先頭30文字（超える場合は「…」で省略）
- ステータスバッジ

選択中のカードはアクセントカラーの左ボーダーとハイライト背景で示す。

### 右ペイン（3モード切替）

右ペインには同時に1つのモードのみ表示する。

| モード | ID | 表示条件 |
|---|---|---|
| 空状態 | `pane-empty` | 起動直後・削除後・ESCキー |
| 詳細 | `pane-detail` | カードクリック・保存後 |
| フォーム | `pane-form` | 新規登録ボタン・編集ボタン |

#### 詳細モードの要素

- 全項目の表示（日付・カテゴリ・金額・メモ・ステータスバッジ・作成日時）
- 編集ボタン → フォームモードへ遷移（既存値をフォームに入力済みで開く）
- 削除ボタン → 確認ダイアログ → 削除実行 → 空状態へ
- ステータス切替ボタン
  - 申請中のとき「精算済にする」ボタンを表示
  - 精算済のとき「申請中に戻す」ボタンを表示
  - クリックで即時更新（サマリと一覧バッジも連動）

#### フォームモードの要素

- 日付（date input、必須）
- カテゴリ（select、必須）
- 金額（number input、必須、最小値1）
- メモ（textarea、任意）
- 保存ボタン・キャンセルボタン
- バリデーション：必須項目未入力・金額0円以下の場合はエラーメッセージを表示して保存を阻止

## 4. CRUD 操作フロー

| 操作 | 起点 | 処理 | 終着 |
|---|---|---|---|
| **登録（Create）** | 新規登録ボタン | フォームモード表示 → 入力 → 保存ボタン → バリデーション → localStorage保存 → 一覧更新 | 詳細モード（登録した経費を表示） |
| **閲覧・一覧（Read）** | アプリ起動時 | localStorageから読込 → 経費カードリスト描画 | 左ペイン一覧表示 |
| **検索（Read）** | 検索ボックスへの入力 | メモ・カテゴリをリアルタイムフィルタリング → 一致カードのみ表示 | 絞り込まれた一覧（検索クリアで全件に戻る） |
| **編集（Update）** | 詳細モードの編集ボタン | フォームモード表示（既存値入り） → 編集 → 保存ボタン → バリデーション → localStorage上書き → 一覧更新 | 詳細モード（編集後の値を表示） |
| **削除（Delete）** | 詳細モードの削除ボタン | 確認ダイアログ → OK → localStorage削除 → 一覧更新 | 空状態 |
| **ステータス切替** | 詳細モードの切替ボタン | status値を `pending` ⇔ `settled` に反転 → localStorage上書き → バッジ・サマリを即時更新 | 詳細モード（更新後のステータスを表示） |

## 5. 永続化方針

| 項目 | 内容 |
|---|---|
| 保存先 | `localStorage`（ブラウザローカル。サーバー不使用） |
| キー | `expense-records` |
| 保存形式 | 経費オブジェクトの配列を `JSON.stringify` した文字列 |
| 読み込み | `JSON.parse(localStorage.getItem('expense-records')) ?? []` |
| 書き込みタイミング | 登録・編集・削除・ステータス変更のたびに配列全体を上書き保存 |
| アクセス方法 | 直接 `localStorage.setItem/getItem` を各所に書かず、`loadRecords()` / `saveRecords(records)` を経由する |

---

## 6. 初期データ

初回起動時（`localStorage.getItem('expense-records')` が `null`）のみ、以下の条件を満たすサンプル5件を自動投入する。

- カテゴリはなるべくバラけさせる（5種以上使用）
- 金額は数百円〜数万円の範囲で変化をつける
- `pending`（申請中）と `settled`（精算済）を混在させる（例: 3件 pending、2件 settled）
- 日付は投入日から直近1〜2ヶ月の範囲

---

## 7. 対象外の機能

以下の機能は今回のスコープに含まない。

| 機能 | 除外理由 |
|---|---|
| レシートOCR（画像からの自動入力） | 外部APIが必要。技術制約に抵触 |
| 承認ワークフロー（申請者／承認者の役割分担） | マルチユーザー機能が必要 |
| クラウド保存・他端末共有 | サーバー・DBが必要。技術制約に抵触 |
| ログイン・マルチユーザー | サーバー認証が必要 |
| 会計ソフト連携・仕訳出力 | 外部APIが必要 |
| CSV / PDF出力 | スコープ外 |
| モバイル対応（レスポンシブ） | PC表示のみを対象とする |
